# XPath 扫描 Skill（专项执行手册）

本文件只讲“怎么做”，默认继承 AGENTS.md 的全部硬约束。

---

## 1. 启动与入口

1. 用户先启动本地 Chrome（9222）并手动登录。
2. 运行 pnpm scan，脚本进入交互模式。
3. 可用命令：scan / url / pages / help / exit。

建议操作节奏：
1. 先 url 确认页面。
2. 再 scan 执行采集。
3. 用户手动切页后重复。

## 2. scan 前检查

执行 scan 前必须检查：
1. 当前 URL 与标题可读取。
2. 非登录页、验证码页、风控页、report 页、about:blank。
3. 页面主要区域已加载，可见可操作。

若命中禁止状态：
1. 立即拒绝扫描。
2. 提示用户先手动处理页面，再重试。

## 3. 元素采集策略

每次仅扫描当前激活页面，按区域切片进行：
1. 顶部栏
2. 导航区
3. 主内容区
4. 表单/筛选区
5. 表格区
6. 分页区
7. 当前 Tab
8. 当前弹窗/抽屉/下拉

采集目标：
1. button / a / input / select / textarea
2. role=button|tab|menuitem|checkbox|radio|switch|combobox
3. 带 onclick 的真实可操作节点
4. 普通 div/section/article/li 形式的可点击卡片：必须具备稳定 data-* 标识和卡片标题，并同时存在框架点击事件或 pointer 光标；名称取卡片标题，禁止依赖动态哈希 class。

过滤条件：
1. 不可见
2. disabled
3. hidden/aria-hidden 区域
4. 纯展示文案
5. 同一点击容器内部重复子节点

表格采集规则：
1. 表格正文中的商品名、商品 ID、订单号、状态、库存数量、金额等动态业务单元格不采集，不得写入元素名称或 XPath。
2. 保留表头全选框、行选择框、操作列控件和表格外分页控件。
3. 操作列依据对应表头“操作/动作”识别，统一标记为“表格行操作”。
4. 表格行操作页面路径增加“表格行”层级，元素名称只保留操作自身文案，例如“编辑库存”。
5. 同一操作在多行重复时只记录一个通用 XPath；允许标记“多元素命中”，实际执行时先用运行时业务条件定位目标行，再在该行内使用操作定位。
6. 通用 XPath 只能使用表格结构、静态表头、控件语义和固定操作文案，不得固化当前行的动态数据。

非表格列表/卡片规则：
1. 同名操作在非标准 table 的列表或卡片中重复出现时，优先使用最近语义容器的稳定静态标题收敛 XPath。
2. 语义容器仅限 section、article、region、dialog 或带稳定测试标识的容器，禁止依赖动态哈希 class。
3. 容器标题包含长编号、日期、金额或超长业务文案时不得写入 XPath。
4. 静态上下文仍不能唯一定位时保留通用 XPath，并标记“多元素命中”；禁止强行使用位置序号。

## 4. 元素命名与类型

元素名称优先来源：
1. 可见文本
2. aria-label
3. placeholder
4. title
5. label 文本

命名要求：
1. 简洁稳定，可读。
2. 去掉动态变量（数量、金额、日期、订单号、ID 等）。
3. 禁止空名称、纯数字、隐私数据。

页面路径与元素名称边界：
1. 页面路径表达真实容器层级和当前视图状态，不表达元素在 DOM 中的出现顺序。
2. 一级导航、二级导航、当前页面、活动 Tab、活动分段/Radio 视图依次进入页面路径。
3. 同级切换项中只有当前活动项进入其内容区域的页面路径，禁止把所有同级项依次拼接。
4. 元素名称只描述元素自身；明确的 placeholder（如“订单编号”）可直接作为名称。
5. 查找字段标签不得越过所属 form、表单项或语义容器，把表单外导航、Tab、切换项当成字段标签。
6. 示例：当前层级为“交易 > 投诉与申诉 > 我要报备 > 全部免责订单”，活动视图为“手工报备”，输入框名称为“订单编号”，则页面路径为“交易 > 投诉与申诉 > 我要报备 > 全部免责订单 > 手工报备 > 订单编号”，元素名称为“订单编号”。

元素类型从固定集合选择：
1. 按钮
2. 链接
3. 输入框
4. 下拉框
5. 多选框
6. 单选框
7. 开关
8. 日期控件
9. Tab
10. 分页
11. 表格行操作
12. 弹窗按钮
13. 抽屉按钮
14. 菜单项
15. 可点击卡片
16. 其他可点击元素

## 5. XPath 生成与校验

生成原则：
1. 优先真实控件节点。
2. 优先稳定语义属性（role、name、placeholder、aria-*、稳定 data-*）。
3. 无稳定属性的表单控件使用所属字段标签和最近表单项生成上下文 XPath；标签匹配应兼容必填星号及中英文冒号。
4. 字段标签 XPath 只能引用静态标签文案，不得引用 input 当前值。
5. 排除 hidden/aria-hidden 区域及其后代。
6. 禁止动态 id、随机 class、绝对 XPath。
7. 禁止用动态值做精确匹配，禁止用 `[1]`、`[2]` 等位置序号伪造唯一性。

写入前必须校验命中：
1. 唯一命中：成功。
2. 多命中可收敛：补上下文后再写。
3. 多命中不可收敛：成功标志写 多元素命中。
4. 未命中：定位方式留空，成功标志写 检索失败。

## 6. 风险操作处理

1. 风险按钮只记录 XPath，不点击。
2. 非风险动作可点击用于展开上下文（如 Tab 切换、分页、展开、下拉）。
3. 引导层优先尝试跳过/关闭；无法关闭时暂停并提示用户手动处理。

## 7. Tab、弹窗、抽屉、下拉

Tab：
1. 逐个切换真实可点击 Tab。
2. 每次切换后重新读取可见 DOM。
3. 采集该 Tab 新增元素。

弹窗/抽屉/下拉：
1. 先记录触发器。
2. 打开后采集内部元素。
3. 关闭并回到原页面。
4. 再次刷新当前可见 DOM。

## 8. 落盘顺序与恢复

每次 scan 完成后按顺序写入：
1. xpath-rows.json
2. xpath-progress.json
3. xpath-result.xlsx

恢复流程：
1. 读取 xpath-progress.json。
2. 读取 xpath-rows.json。
3. 连接 Chrome。
4. 打印当前 URL 与标题。
5. 等待用户输入 scan 继续。

## 9. 去重规则

唯一键：页面路径 / 元素名称 / 元素类型 / 适用流程。

策略：
1. 同键同 XPath：跳过。
2. 同键新 XPath 更稳定：更新。
3. 同一点击容器的图标/文字重复：只保留主点击容器。

## 10. 失败与重试

1. 单元素最多重试 2 次。
2. 重试前刷新当前可见 DOM。
3. 仍失败则保留记录并写失败原因。
4. 不因单元素失败中断整页。

## 11. 页面完成判定

以下完成后才算页面完成：
1. 首屏与滚动区已检查。
2. 当前 Tab 已检查。
3. 非风险弹层已检查。
4. 结果已写入 JSON + XLSX。
5. 进度已更新。

## 12. 最小实现检查清单

1. 能连接本地 Chrome。
2. 有交互命令 scan/url/pages/help/exit。
3. 只在用户输入 scan 时扫描。
4. 生成并校验 XPath。
5. 输出满足固定 7 列。
6. 支持实时落盘、去重、恢复。

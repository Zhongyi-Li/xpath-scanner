# 独立补扫命令设计

## 目标

为交互式 XPath 扫描器增加 `rescan --<文件名>.xlsx` 命令。该命令只扫描当前激活页面，并把本次结果写入一份独立 Excel，供遗漏页面补扫使用。

## 命令行为

- 用户先通过 `pnpm scan` 进入现有交互模式。
- 用户在 Chrome 中手动打开并激活需要补扫的业务页面。
- 用户输入 `rescan --漏扫页面.xlsx`。
- 扫描器沿用普通 `scan` 的页面安全检查、元素采集、XPath 生成和校验规则。
- 输出文件位于项目根目录，仅包含本次当前页面采集到的记录。
- 输出 Sheet 固定为 `XPath清单`，列固定为现有 7 列，平台固定为 `天猫`。

## 数据隔离

`rescan` 不读取、不修改以下正式数据文件：

- `xpath-result.xlsx`
- `xpath-rows.json`
- `xpath-progress.json`

补扫不会参与正式数据的去重、合并或进度更新。调试信息仍可按项目约束追加到 `xpath-debug.log`。

## 文件名与覆盖保护

- 参数必须使用 `--<文件名>.xlsx` 格式。
- 文件名不得为空，扩展名必须为 `.xlsx`，扩展名大小写均可识别。
- 只接受文件名，不接受目录、绝对路径或路径穿越片段，确保输出始终位于项目根目录。
- 禁止使用正式结果文件名 `xpath-result.xlsx`。
- 目标文件已经存在时拒绝覆盖，并提示用户更换文件名。
- 保留用户输入的文件名大小写，不因命令解析而转成小写。

## 错误处理

- 当前没有可用页面、页面未加载或未采集到元素时，不创建文件。
- 登录页、验证码页、风控页、Playwright 报告页等禁止页面继续拒绝扫描。
- 参数格式错误时显示正确示例，不扫描、不落盘。
- Excel 写入失败时报告错误；由于该模式不维护 JSON 或进度文件，不产生任何正式数据变更。

## 代码结构

- 从交互输入中分离命令名和原始参数，保证文件名大小写不丢失。
- 提取并复用 Excel 工作簿写入逻辑，使普通扫描仍写入固定正式文件，补扫写入经校验的独立目标文件。
- 将补扫参数解析和输出路径校验放在独立、可单元测试的模块中。

## 测试

单元测试覆盖：

- 正确解析 `rescan --漏扫页面.xlsx`。
- 保留文件名大小写。
- 拒绝缺少参数、非 `.xlsx` 扩展名、目录路径、路径穿越和正式结果文件名。
- 已存在文件的覆盖保护由执行层验证。
- 完整测试与 TypeScript 类型检查确保原有扫描行为不回归。

# 上下文感知 XPath 设计

## 目标

减少非表格列表操作和无稳定属性表单控件的 XPath 多元素命中，同时遵守不使用动态业务值、敏感值和不稳定序号的约束。

## 根因

1. 无 `name`、`placeholder`、`aria-*` 等稳定属性的输入框会退化为通用 DOM 结构 XPath，多个表单项因此命中同一组控件。
2. 非标准表格实现的卡片/列表会重复渲染同名操作按钮，现有表格操作列规则无法识别这类容器。

## 设计

### 表单字段上下文

扫描器在生成候选 XPath 时复用已识别的 `fieldLabel`。优先限定在同时包含字段标签和目标控件的最近语义表单项内，再定位真实的 `input`、`textarea`、`select` 或组合框节点。

候选只能使用静态字段标签，不得使用控件当前值。字段标签候选必须经过实时命中计数，只有唯一命中时才由既有选择器标记为“成功”。

### 非表格列表/卡片上下文

对重复操作按钮向上查找最近的语义容器。容器可使用稳定 `data-*`、静态标题、`role=listitem`、`li`、`article` 等语义构造候选 XPath。

若容器上下文包含疑似动态仓库名、编号、金额、日期或其他业务值，禁止写入 XPath。无法在静态上下文中唯一收敛时，继续输出通用 XPath并标记“多元素命中”，不得强行加入位置序号。

### 候选优先级

1. 现有稳定语义属性候选。
2. 字段标签限定候选。
3. 非表格列表/卡片静态上下文候选。
4. 现有通用文本和结构候选。

最终仍由 `selectStableXPath` 选择第一个唯一命中的候选；没有唯一候选时保持现有安全标记。

## 文件边界

- 新增 `src/xpath-context.ts`：纯函数生成字段标签和列表容器上下文 XPath，便于单元测试。
- 修改 `src/scan-current-page.ts`：采集必要 DOM 上下文，合并候选并计算命中数。
- 修改 `tests/element-semantics.test.ts` 或新增专项测试：覆盖标签收敛、静态卡片上下文和动态上下文拒绝。
- 修改 `skills/xpath-scan/SKILL.md`：记录非表格列表及字段标签定位规则。

## 验收

1. 相同结构的“仓库名称、仓库别名、邮政编码、联系人、联系电话”等控件可通过各自字段标签形成唯一 XPath。
2. 非表格重复操作仅在存在稳定静态容器上下文时标记成功。
3. 不使用 `[1]`、`[2]` 等位置序号，不使用输入值或动态业务名称。
4. 既有表格、iframe、日期区间和可点击卡片规则不回归。
5. `pnpm test:unit` 和 TypeScript 检查通过。
